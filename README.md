# TriMatrix Digital Marketing Suite

TriMatrix is a state-of-the-art, fully functional digital marketing suite designed to automate, analyze, and optimize marketing workflows.

## Architecture & Codebase Guide

The application is built using **React 18**, **Vite**, **Tailwind CSS**, and **TypeScript**. It features a modern, glassy-neumorphic UI design.

### Key Directories & Files

- `/src/pages/`: Contains the main views of the application.
  - `Dashboard.tsx`: The main overview dashboard with charts and metrics.
  - `Workflow.tsx`: The visual Workflow Builder using `@xyflow/react`. Allows users to drag-and-drop nodes to create marketing automations.
  - `AI.tsx`: The AI Assistant and Customer Persona Generator.
  - `SeoEngine.tsx`: SEO tools including keyword tracking and competitor gap analysis.
  - `ContentGen.tsx`: Multi-platform AI content generation and text summarization.
  - `Automation.tsx`: Automation rules dashboard and performance metrics.
- `/src/components/`: Reusable UI components (e.g., Sidebar, Header, ErrorBoundary).
- `/src/App.tsx`: Main application routing and layout wrapper.
- `/src/index.css`: Global Tailwind CSS configuration and custom variables.

## Database Schema

Below is the proposed relational database schema for the entire application. This schema is designed to support users, workflows, SEO data, generated content, and analytics.

### `users`
Stores user account information and preferences.
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | UUID (PK) | Unique identifier for the user. |
| `email` | VARCHAR | User's email address. |
| `password_hash` | VARCHAR | Hashed password. |
| `role` | VARCHAR | User role (e.g., 'admin', 'user'). |
| `created_at` | TIMESTAMP | Account creation date. |

### `workflows`
Stores the visual workflows created in the Workflow Builder.
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | UUID (PK) | Unique identifier for the workflow. |
| `user_id` | UUID (FK) | Reference to `users.id`. |
| `name` | VARCHAR | Name of the workflow. |
| `nodes_json` | JSONB | JSON representation of React Flow nodes. |
| `edges_json` | JSONB | JSON representation of React Flow edges. |
| `is_active` | BOOLEAN | Whether the workflow is currently active. |
| `last_executed` | TIMESTAMP | Timestamp of the last execution. |
| `created_at` | TIMESTAMP | Workflow creation date. |

### `workflow_executions`
Logs the execution history of workflows.
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | UUID (PK) | Unique identifier for the execution log. |
| `workflow_id` | UUID (FK) | Reference to `workflows.id`. |
| `status` | VARCHAR | Status ('success', 'failed', 'running'). |
| `output_log` | TEXT | Detailed log of the execution. |
| `executed_at` | TIMESTAMP | When the execution occurred. |

### `seo_keywords`
Stores tracked keywords for the SEO Engine.
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | UUID (PK) | Unique identifier for the keyword record. |
| `user_id` | UUID (FK) | Reference to `users.id`. |
| `keyword` | VARCHAR | The search term. |
| `volume` | INTEGER | Monthly search volume. |
| `difficulty` | INTEGER | Keyword difficulty score (0-100). |
| `trend` | VARCHAR | Trend percentage (e.g., '+12%'). |
| `status` | VARCHAR | Tracking status ('Active', 'Monitoring'). |

### `generated_content`
Stores content generated by the AI Content Generation system.
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | UUID (PK) | Unique identifier for the content. |
| `user_id` | UUID (FK) | Reference to `users.id`. |
| `topic` | VARCHAR | The topic or prompt used. |
| `platform` | VARCHAR | Target platform (e.g., 'blog', 'twitter'). |
| `tone` | VARCHAR | Tone of voice used. |
| `content_text` | TEXT | The actual generated content. |
| `created_at` | TIMESTAMP | When the content was generated. |

### `customer_personas`
Stores AI-generated customer personas.
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | UUID (PK) | Unique identifier for the persona. |
| `user_id` | UUID (FK) | Reference to `users.id`. |
| `name` | VARCHAR | Name of the persona. |
| `demographics` | JSONB | Age, location, occupation, etc. |
| `psychographics` | JSONB | Interests, values, lifestyle. |
| `behavioral` | JSONB | Buying habits, brand interactions. |
| `created_at` | TIMESTAMP | When the persona was created. |

### `integrations`
Stores API keys and OAuth tokens for third-party services (encrypted).
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | UUID (PK) | Unique identifier. |
| `user_id` | UUID (FK) | Reference to `users.id`. |
| `service_name` | VARCHAR | Name of the service (e.g., 'twitter', 'ga4'). |
| `api_key` | VARCHAR | Encrypted API key or token. |
| `connected_at` | TIMESTAMP | When the integration was established. |

## How to Make Changes

1. **Adding a New Page**: Create a new `.tsx` file in `/src/pages/`, then add a route in `/src/App.tsx` and a link in `/src/components/Sidebar.tsx`.
2. **Adding a New Workflow Node**: Open `/src/pages/Workflow.tsx`, locate the `PREMADE_NODES` array, and add a new object with `type`, `label`, `icon`, `color`, `border`, and `description`.
3. **Modifying Styles**: The app uses Tailwind CSS. You can adjust global colors and theme settings in `/src/index.css`.
4. **State Management**: Currently using React `useState` and `useEffect`. For larger scale, consider adding Redux or Zustand.
